# 📂 โน้ตบุ๊กศูนย์รวมข้อมูลระบบ (Project Context & Docs)

ยินดีต้อนรับสู่ **"สมองส่วนกลาง"** ของโปรเจกต์เรา! โฟลเดอร์นี้สร้างขึ้นมาเพื่อเก็บรวบรวม **Context, ข้อมูลดิบ, แผนงาน และกฎเกณฑ์ทั้งหมด** ก่อนที่เราจะเริ่มลงมือเขียนโค้ดจริง (Day 0) 

เป้าหมายสูงสุดคือเพื่อให้เราทั้งสองคน มองเห็นภาพสถาปัตยกรรมแอปพลิเคชันตรงกัน 100% โดยไม่ต้องเดา

> ⚠️ **กฎเหล็กประจำเป็นแผนก:** 
> 1. ห้ามลบไฟล์ในโฟลเดอร์นี้เด็ดขาด ถ้าต้องการเปลี่ยนแปลงเชิงโครงสร้าง รบกวนเปิดประเด็นคุยกันในแชตก่อนตลอด
> 2. ทุกครั้งที่อัปเดตระบบ หรือปรับเปลี่ยนแผนงาน ให้มาบันทึกที่ **Activity Log** ด้านล่างสุดของไฟล์นี้ด้วย

## 🗺️ แผนผังเอกสาร (File Directory)

นี่คือรายชื่อไฟล์ในโฟลเดอร์นี้และหน้าที่ของมันเปิดอ่านตามหมวดหมู่ได้เลย:

### 🎯 1. Product & Business Strategy
*   **`prd.md` (Product Requirement Document):** คัมภีร์หลักของแอป สรุปว่าแอปเราคืออะไร แก้ปัญหาอะไร และฟีเจอร์ไหนที่เป็น **MVP (Must-Have)** ที่ต้องทำในเฟสแรก
*   **`user_journey.md`:** เส้นทางการเดินหน้าจอของผู้ใช้งาน ตั้งแต่เริ่มกดสมัครสมาชิกไปจนถึงปิดแอป

### 🏗️ 2. Technical Architecture & Data
*   **`architecture.md`:** แผนผังการเชื่อมต่อระบบ เทคโนโลยี Stack ที่เราเลือกใช้ และการจัดการ Infrastructure (เช่น Frontend คุยกับ Backend อย่างไร วางระบบไว้ที่ไหน)
*   **`database_schema.md`:** พิมพ์เขียวตารางข้อมูล (Database Tables & Relationships) ออกแบบโครงสร้างข้อมูลทั้งหมดร่วมกันไว้ที่นี่
*   **`api_spec.md`:** เอกสารข้อตกลงการรับ-ส่งข้อมูลระหว่างหน้าบ้านและหลังบ้าน

### 📜 3. Team Governance & Standards
*   **`coding_standards.md`:** กฎการเขียนโค้ด รูปแบบการตั้งชื่อตัวแปร ฟอร์แมตของไฟล์ เพื่อให้โค้ดที่เราทั้งคู่และ Claude Code เขียนออกมาเป็นสไตล์เดียวกันอ่านง่าย
*   **`git_flow.md`:** กฎการแตกกิ่ง (Branch) บน GitHub และการทำ Code Review ก่อนกด Merge งานเข้ากิ่งหลัก

## 🤖 วิธีใช้โฟลเดอร์นี้สั่งงาน Claude Code ให้เต็มประสิทธิภาพ

โฟลเดอร์นี้คือขุมทรัพย์ของ AI เวลาที่เราจะสั่งให้ Claude Code ช่วยเขียนโค้ด ฟีเจอร์ใหม่ หรือทำ API **ห้ามพิมพ์สั่งปากเปล่า** ให้ใช้คำสั่งอ้างอิงไฟล์ในนี้เสมอ เช่น:

> *"Claude, รบกวนอ่านกฎใน `docs/coding_standards.md` และโครงสร้างฐานข้อมูลใน `docs/database_schema.md` ก่อน จากนั้นค่อยช่วยเจนโค้ดสำหรับระบบสมัครสมาชิกให้หน่อย"*

การทำแบบนี้จะทำให้ Claude ทำงานได้ฉลาดขึ้นอย่างก้าวกระโดด โค้ดไม่หลุดโฟกัส และแทบจะไม่ต้องเสียเวลานั่งแก้บั๊กซ้ำซ้อนเลย
